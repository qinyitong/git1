1. 创建版本库(自己创建一个文件，右键git bash)::
	git init                                           //把这个目录变成git可以管理的仓库

	git add *.c                                      //将目录下以 .c 结尾的添加到暂存区里面去
	1.git add  readme.txt                       //将新增/修改文件 添加到暂存区里面去
	git add README hello.php           
	2.git commit -m 'readme.txt提交'    //把文件提交到仓库 ‘注释’（可以使用git commit一次性提交到分支上）
	git status                                        //来查看是否还有文件未提交

	git diff readme.txt                         //看下readme.txt文件到底改了什么内容

2.版本回退：：
	git log                                             //查看下修改历史记录 （版本号）
	git log Cpretty=oneline                  //查看简明修改历史记录

	git reset --hard HEAD^                   //回退到上个版本
	git reset --hard HEAD^^                 //回退到上上个版本
	git reset --hard HEAD~100              //回退到上100个版本
	git reset --hard 版本号                      //回退到某个版本（参数：6fcfc89）

3.撤销修改和删除文件：：
	1）git reset --hard HEAD^
	2）git checkout -- readme.txt           //把readme.txt文件在工作区做的修改全部撤销，-- 很重要，如果没有 -- 的话，那么命令变成创建分支了

	rm b.txt                                              //删掉文件b.txt

5.远程仓库：：
	
	git remote add origin https://github.com/qinyitong/git1.git
	git push -u origin master                    //把本地仓库分支master内容推送到元仓库（把本地库的内容推送到远程，使用 git push命令，实际上是把当前分支master推送到远程。第一次推送master分支时，加上了 Cu参数，Git不但会把本地的master分支内容推送的远程新的master分支，还会把本地的master分支和远程的master分支关联起来，在以后的推送或者拉取时就可以简化命令。推送成功后，可以立刻在github页面中看到远程库的内容已经和本地一模一样了，）    
	git push origin master                          //从现在起，只要本地作了提交，就可以通过命令提交     

	git clone https://githua.com/qinyitong/git2     //克隆 、

	git clone ssh://example.com/~/www/project.git   //（从服务器克隆一个库）
	git push ssh://example.com/~/www/project.git   //修改之后可以进行推送到服务器。
	
	git clone <repo>                                  //从现有 Git 仓库中拷贝项目
	 git clone <repo> <directory>             //克隆到指定的目录  （repo:Git 仓库；directory:本地目录 ）     
6.创建与合并分支：：
	git cheakout dev                                   //创建dev分支
	git cheakout -b dev                               //创建并切换分支dev
	git branch                                              //查看当前的分支（查看分支，会列出所有的分支）

	git merge dev                                        //在master分支上合并dev分支内容

	git branch -d dev                                   //删除dev分支

	cat readme.txt                                       //查看readme.txt 文件内容

	git remote                                               //查看远程库的信息 
	git remote   -v                                         //查看远程库的详细信息 

	git push origin master                            //推送分支
